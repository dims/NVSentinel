# Quick Start: NVSentinel with PostgreSQL

This guide helps you quickly set up NVSentinel using PostgreSQL as the datastore with certificate-based authentication.

## Prerequisites

- PostgreSQL 12+ server running with TLS support
- Database admin access
- NVSentinel deployment environment (Kubernetes or Docker)
- cert-manager installed in Kubernetes (for automatic certificate generation)

## Step 1: Database Setup

### 1.1 Create Database and User

```bash
# Connect to PostgreSQL as admin
psql -h your-postgres-host -U postgres

# Create database and user for certificate-based authentication
CREATE DATABASE nvsentinel;
CREATE USER postgresql;  # Username matches certificate CommonName
GRANT ALL PRIVILEGES ON DATABASE nvsentinel TO postgresql;

# Connect to the nvsentinel database
\c nvsentinel

# Grant permissions
GRANT ALL ON SCHEMA public TO postgresql;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO postgresql;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO postgresql;

\q
```

### 1.2 Configure PostgreSQL for Certificate Authentication

Edit `postgresql.conf`:
```conf
# Enable SSL
ssl = on
ssl_cert_file = '/path/to/server.crt'
ssl_key_file = '/path/to/server.key'
ssl_ca_file = '/path/to/ca.crt'
```

Edit `pg_hba.conf`:
```conf
# Certificate-based authentication
hostssl nvsentinel postgresql 0.0.0.0/0 cert clientcert=verify-full
```

### 1.3 Test Connection

```bash
# Test the connection with certificate
psql "host=your-postgres-host port=5432 dbname=nvsentinel user=postgresql sslmode=require sslcert=/path/to/client.crt sslkey=/path/to/client.key sslrootcert=/path/to/ca.crt"
```

## Step 2: Configure NVSentinel

### Kubernetes with Certificate Authentication (Recommended)

NVSentinel uses cert-manager to automatically generate and manage certificates for PostgreSQL authentication.

```yaml
# values-postgresql.yaml
global:
  datastore:
    provider: "postgresql"
    connection:
      host: "nvsentinel-postgresql.nvsentinel.svc.cluster.local"
      port: 5432
      database: "nvsentinel"
      username: "postgresql"
      sslmode: "require"
      sslcert: "/etc/ssl/client-certs/tls.crt"
      sslkey: "/etc/ssl/client-certs/tls.key"
      sslrootcert: "/etc/ssl/client-certs/ca.crt"
    options:
      maxConnections: "25"
      maxIdleConnections: "10"
      connectionMaxLifetime: "1h"
      pollInterval: "5s"

# Enable built-in PostgreSQL with TLS
postgresql:
  enabled: true
  architecture: standalone
  auth:
    username: "postgresql"
    database: "nvsentinel"
  tls:
    enabled: true
    autoGenerated: false
    certificatesSecret: "postgresql-server-cert"
```

### Environment Variables (Alternative)

```bash
export DATASTORE_PROVIDER=postgresql
export DATASTORE_HOST=your-postgres-host
export DATASTORE_PORT=5432
export DATASTORE_DATABASE=nvsentinel
export DATASTORE_USERNAME=postgresql
export DATASTORE_SSLMODE=require
export DATASTORE_SSLCERT=/etc/ssl/client-certs/tls.crt
export DATASTORE_SSLKEY=/etc/ssl/client-certs/tls.key
export DATASTORE_SSLROOTCERT=/etc/ssl/client-certs/ca.crt
```

### Manual Certificate Configuration

If not using cert-manager, mount certificates manually:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: nvsentinel-datastore-config
data:
  DATASTORE_PROVIDER: "postgresql"
  DATASTORE_HOST: "your-postgres-host"
  DATASTORE_PORT: "5432"
  DATASTORE_DATABASE: "nvsentinel"
  DATASTORE_USERNAME: "postgresql"
  DATASTORE_SSLMODE: "require"
  DATASTORE_SSLCERT: "/etc/ssl/client-certs/tls.crt"
  DATASTORE_SSLKEY: "/etc/ssl/client-certs/tls.key"
  DATASTORE_SSLROOTCERT: "/etc/ssl/client-certs/ca.crt"
---
apiVersion: v1
kind: Secret
metadata:
  name: postgresql-client-cert
type: kubernetes.io/tls
data:
  tls.crt: <base64-encoded-client-certificate>
  tls.key: <base64-encoded-client-private-key>
  ca.crt: <base64-encoded-ca-certificate>
```

## Step 3: Deploy NVSentinel

### Kubernetes Deployment with Built-in PostgreSQL

Deploy NVSentinel with the built-in PostgreSQL chart:

```bash
# Deploy with PostgreSQL support
helm install nvsentinel ./distros/kubernetes/nvsentinel \
  -f ./distros/kubernetes/nvsentinel/values-tilt-postgresql.yaml \
  --set global.tiltMode=false
```

### Development Environment

For development with Tilt:

```bash
# Clone the repository
git clone <nvsentinel-repo>
cd nvsentinel

# Set the PostgreSQL Tiltfile
export TILTFILE=Tiltfile.postgresql

# Start development environment
make dev-env

# Or just run Tilt
make tilt-up
```

### Production Kubernetes Deployment

For production environments:

```yaml
# values-production-postgresql.yaml
global:
  datastore:
    provider: "postgresql"
    connection:
      host: "your-external-postgres-host"
      port: 5432
      database: "nvsentinel"
      username: "postgresql"
      sslmode: "require"
      sslcert: "/etc/ssl/client-certs/tls.crt"
      sslkey: "/etc/ssl/client-certs/tls.key"
      sslrootcert: "/etc/ssl/client-certs/ca.crt"

# Disable built-in PostgreSQL for production
postgresql:
  enabled: false

# Configure client certificate mounting for all components
platformConnector:
  postgresqlStore:
    enabled: true
    clientCertMountPath: "/etc/ssl/client-certs"

fault-quarantine:
  clientCertMountPath: "/etc/ssl/client-certs"

fault-remediation:
  clientCertMountPath: "/etc/ssl/client-certs"

node-drainer:
  clientCertMountPath: "/etc/ssl/client-certs"

health-events-analyzer:
  clientCertMountPath: "/etc/ssl/client-certs"
```

Deploy with:
```bash
helm install nvsentinel ./distros/kubernetes/nvsentinel \
  -f values-production-postgresql.yaml
```

## Step 4: Verify Setup

### 4.1 Check Certificate Configuration

Verify that certificates are properly mounted:

```bash
# Check certificate secret
kubectl get secret postgresql-client-cert -n nvsentinel

# Check if cert-manager created certificates
kubectl get certificates -n nvsentinel

# Verify certificate details
kubectl describe certificate postgresql-client-cert -n nvsentinel
```

### 4.2 Check Database Schema

The PostgreSQL provider automatically creates the required tables:

```bash
# For built-in PostgreSQL
kubectl exec -it nvsentinel-postgresql-0 -n nvsentinel -- \
  psql -U postgresql -d nvsentinel -c "\dt"

# For external PostgreSQL
psql "host=your-postgres-host port=5432 dbname=nvsentinel user=postgresql sslmode=require sslcert=/path/to/client.crt sslkey=/path/to/client.key sslrootcert=/path/to/ca.crt" -c "\dt"
```

You should see:
- `maintenance_events`
- `health_events`
- `datastore_changelog`
- `resume_tokens`

### 4.3 Check NVSentinel Logs

```bash
# Check platform connector logs
kubectl logs -l app.kubernetes.io/name=platform-connectors -n nvsentinel

# Check fault module logs
kubectl logs -l app.kubernetes.io/name=fault-quarantine -n nvsentinel
kubectl logs -l app.kubernetes.io/name=fault-remediation -n nvsentinel
kubectl logs -l app.kubernetes.io/name=node-drainer -n nvsentinel
```

Look for log messages like:
```
I1021 10:00:00.123456 1 datastore.go:45] Successfully connected to PostgreSQL database: nvsentinel-postgresql.nvsentinel.svc.cluster.local
I1021 10:00:00.234567 1 datastore.go:123] Successfully created PostgreSQL tables and indexes
I1021 10:00:00.345678 1 ssl.go:89] Using SSL certificate authentication for PostgreSQL
```

### 4.4 Test Data Flow

1. **Check PostgreSQL connectivity**:
   ```bash
   # Port-forward to PostgreSQL
   kubectl port-forward svc/nvsentinel-postgresql 5432:5432 -n nvsentinel &

   # Test connection (if you have client certificates locally)
   psql "host=localhost port=5432 dbname=nvsentinel user=postgresql sslmode=require"
   ```

2. **Check the database**:
   ```sql
   SELECT COUNT(*) FROM health_events;
   SELECT COUNT(*) FROM maintenance_events;
   ```

3. **Monitor change streams**:
   ```sql
   SELECT table_name, COUNT(*) FROM datastore_changelog GROUP BY table_name;
   ```

## Step 5: Production Considerations

### Security

NVSentinel PostgreSQL integration uses certificate-based authentication for enhanced security:

```bash
# Certificate-based authentication is automatically configured
# Certificates are managed by cert-manager with automatic rotation

# For external PostgreSQL, ensure proper TLS configuration
export DATASTORE_SSLMODE=require
export DATASTORE_SSLCERT=/etc/ssl/client-certs/tls.crt
export DATASTORE_SSLKEY=/etc/ssl/client-certs/tls.key
export DATASTORE_SSLROOTCERT=/etc/ssl/client-certs/ca.crt

# Ensure certificates have appropriate permissions
chmod 640 /etc/ssl/client-certs/tls.key
```

## Troubleshooting

### Common Issues

1. **Connection refused**
   ```bash
   # Check if PostgreSQL is running
   pg_isready -h your-postgres-host -p 5432

   # Check firewall rules
   telnet your-postgres-host 5432
   ```

2. **Permission denied**
   ```sql
   -- Grant missing permissions
   GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO nvsentinel_user;
   GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO nvsentinel_user;
   ```

3. **SSL connection issues**
   ```bash
   # Test SSL connection
   psql "host=your-postgres-host port=5432 dbname=nvsentinel user=nvsentinel_user sslmode=require"
   ```

4. **Performance issues**
   ```sql
   -- Check active connections
   SELECT count(*) FROM pg_stat_activity WHERE datname = 'nvsentinel';

   -- Check slow queries
   SELECT query, mean_time, calls FROM pg_stat_statements ORDER BY mean_time DESC;
   ```

### Getting Help

1. Check NVSentinel logs with increased verbosity:
   ```bash
   # Set log verbosity for more detailed output
   kubectl set env deployment/nvsentinel-platform-connector KLOG_V=2 -n nvsentinel
   ```

2. Check PostgreSQL logs:
   ```bash
   # For built-in PostgreSQL
   kubectl logs nvsentinel-postgresql-0 -n nvsentinel

   # For external PostgreSQL
   tail -f /var/log/postgresql/postgresql-*.log
   ```

3. Test database connection manually:
   ```bash
   # Using client certificates
   kubectl exec -it nvsentinel-postgresql-0 -n nvsentinel -- \
     psql -U postgresql -d nvsentinel

   # Debug certificate issues
   kubectl exec -it deployment/nvsentinel-platform-connector -n nvsentinel -- \
     ls -la /etc/ssl/client-certs/
   ```

4. Debug certificate authentication:
   ```bash
   # Check certificate validity
   openssl x509 -in /etc/ssl/client-certs/tls.crt -text -noout

   # Check certificate chain
   openssl verify -CAfile /etc/ssl/client-certs/ca.crt /etc/ssl/client-certs/tls.crt
   ```
